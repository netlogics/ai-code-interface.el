#+TITLE: Repository Map

* Purpose
This document is a lightweight navigation guide for quickly reading this repository. It is meant for new human contributors and AI coding agents, and it should be reviewed by maintainers because several directory purposes are inferred from source layout and tests rather than explicitly documented.

* What This Repository Does
- Provides a uniform Emacs interface for multiple AI coding backends through [[file:../../ai-code.el::ai-code-menu][ai-code-menu]] and the backend registry in [[file:../../ai-code-backends.el::ai-code-backends][ai-code-backends]].
- Runs native CLI backends through shared terminal/session infrastructure in [[file:../../ai-code-backends-infra.el::ai-code-backends-infra--start-cli-session][ai-code-backends-infra--start-cli-session]] and adapter files such as [[file:../../ai-code-codex-cli.el::ai-code-codex-cli][ai-code-codex-cli]].
- Builds context-aware prompts for code changes, questions, TODO implementation, git review, prompt files, task files, and harness follow-ups via modules such as [[file:../../ai-code-change.el::ai-code-code-change][ai-code-change]], [[file:../../ai-code-discussion.el::ai-code-ask-question][ai-code-discussion]], [[file:../../ai-code-prompt-mode.el::ai-code--insert-prompt][ai-code-prompt-mode]], and [[file:../../ai-code-harness.el::ai-code-auto-test-type][ai-code-harness]].
- Exposes Emacs editor/project state to agents through MCP tools in [[file:../../ai-code-mcp-server.el::ai-code-mcp-builtins-setup][ai-code-mcp-server]] and backend launch adapters in [[file:../../ai-code-mcp-agent.el::ai-code-mcp-agent-prepare-launch][ai-code-mcp-agent]].
- Ships ERT tests, prompt harness files, snippets, examples, CI workflows, and generated/user-facing documentation alongside the package.

* Suggested Reading Order
1. Start with [[file:../../README.org::1][README.org]] to understand the user-facing workflows, supported backends, dependencies, and keybinding model.
2. Read [[file:../../AGENTS.md::1][AGENTS.md]] for repository-local coding and verification expectations.
3. Read [[file:../../ai-code.el::ai-code-menu][ai-code-menu]] and the surrounding transient groups in [[file:../../ai-code.el::1][ai-code.el]] to see the public command surface.
4. Read [[file:../../ai-code-backends.el::ai-code-backends][ai-code-backends]] and [[file:../../ai-code-backends.el::ai-code--apply-backend][ai-code--apply-backend]] to understand backend selection and dispatch.

Read the prompt-building path: [[file:../../ai-code-change.el::ai-code-code-change][ai-code-code-change]], [[file:../../ai-code-discussion.el::ai-code-ask-question][ai-code-ask-question]], [[file:../../ai-code-prompt-mode.el::ai-code--insert-prompt][ai-code--insert-prompt]], and [[file:../../ai-code-utils.el::ai-code--get-context-files-string][ai-code--get-context-files-string]].
7. Read [[file:../../ai-code-agile.el::ai-code--refactoring-techniques-catalog][ai-code--refactoring-techniques-catalog]] and [[file:../../ai-code-harness.el::ai-code--resolve-auto-test-suffix-for-send][ai-code--resolve-auto-test-suffix-for-send]] if editing refactoring, TDD, test-after-change, or follow-up prompts.
